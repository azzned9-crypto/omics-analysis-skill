# 单细胞 RNA-seq 下游分析模块

## 定位

这个 skill 承接 `omics-qc-process` 输出的 clean rds/h5ad 文件，负责单细胞转录组的下游分析全流程。

核心流程：数据合并 → 整合（批次校正）→ 聚类 → Marker Gene 识别 → 细胞类型注释 → 差异表达分析 → 高级分析（拟时序、细胞通讯、干性预测）。

不负责 QC（由 `omics-qc-process` 负责），不负责上游 fastq 处理（由 DNBC4tools / Cell Ranger 等上游软件负责）。

## 前置条件

必须先明确：

1. QC 已通过，每样本 clean rds/h5ad 可用。
2. 样本 metadata 完整（样本名、分组、时间点、batch）。
3. 组织类型已明确。
4. 参考基因组版本已确认。
5. 物种已确认（人/鼠/其他）。

缺少任一项时都应停止。

## 必问项

以下问题属于"必须问清楚，否则容易方法错误"的问题：

1. 组织类型是什么？例如外周血 PBMC、肿瘤组织、脑组织、肠组织、肝组织、肺组织、心脏组织等。不同组织的细胞类型组成和 marker 完全不同。
2. 物种是什么？人、鼠还是其他物种？影响参考数据集选择和线粒体 pattern。
3. 是否有预期的细胞类型或 marker gene 列表？如果有，注释阶段优先用用户提供的 marker 进行验证。
4. 样本是否来自多个 batch？是否需要整合？多个 batch 不整合会导致聚类结果被批次效应主导。
5. 倾向的整合方法？Harmony（快速、适用性广）、Seurat CCA（经典方法）、Seurat RPCA（快速版 CCA）、scVI（深度学习）。若不确定，默认 Harmony。
6. 聚类分辨率偏好？或接受默认扫描多个分辨率（0.3、0.5、0.8、1.0）后选择最优。
7. 细胞类型注释方式？手动 marker 验证、SingleR（参考数据集）、CellTypist（预训练模型）、Azimuth（Seurat 官方参考）。若不确定，默认手动 + SingleR 交叉验证。
8. 是否需要拟时序分析？若需要，倾向 Monocle3 还是 CellRank？
9. 是否需要细胞通讯分析？若需要，使用 CellChat。
10. 是否需要干性/分化潜能预测？若需要，使用 CytoTRACE2。
11. 差异分析层面？cluster 间 marker（FindAllMarkers）、条件间 DEG（within cell type）、样本间比较。还是多种都做？

## 必须输出

最少输出以下内容：

1. 整合后的 Seurat/AnnData 对象（rds/h5ad）。
2. 聚类结果（含 UMAP/t-SNE 坐标和 cluster 标签）。
3. 细胞类型注释结果（每个 cluster 对应的 cell type）。
4. 每个 cluster 的 marker gene 列表（CSV）。
5. 差异表达结果表（若适用）。
6. 关键可视化图表（UMAP、dotplot、heatmap、featureplot 等）。
7. 参数与决策记录（整合方法、分辨率、注释策略等）。

## 风险样本规则

1. 细胞数过少的样本在整合中可能被过度校正或丢失特征，需要标记。
2. 批次效应过强时整合方法选择影响结果解读，需要在报告中说明。
3. 注释结果与预期 marker 不一致时需标记并记录，不能直接覆盖用户预期。
4. 整合后某些样本的细胞特征可能被弱化，需要检查每样本在整合前后的分布变化。
5. 聚类分辨率过高或过低都会影响结果，需要多分辨率扫描比较。

## 停止项

遇到以下情况必须停止：

1. QC 未通过或 clean data 不可用。
2. 组织类型不明确，无法选择参考 marker。
3. 样本数或细胞数不足以支撑统计效力。
4. metadata 缺失关键分组信息。
5. 物种不明确，无法选择参考数据集和线粒体 pattern。

## 核心原则

这个 skill 的任务是：

- 先确认生物学背景（组织类型、预期细胞类型），再选择分析策略。
- 整合方法选择必须基于批次结构，不能盲目套用。
- 注释结果必须经过 marker 验证，不能仅依赖自动注释。
- 参数和决策必须记录，支持结果复现。
- 不同组织类型的 marker 和注释策略不同，不能通用。
