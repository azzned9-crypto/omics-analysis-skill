# 组学可视化模块

## 目标

用于组学结果的出图、修图和统一风格。它负责把已有结果表、矩阵或用户提供的绘图代码整理成论文、汇报或检查用的图。

当用户只想画图、修图、统一风格时，使用这个模块。

这个模块不负责上游统计分析本身。

## 默认行为

- 优先使用 R。
- 优先使用 `ggplot2` 和可复现的数据整理流程。
- 连续变量默认使用 `viridis`。
- 默认使用 `theme_bw()` 或项目统一主题。
- 默认同时导出 PDF 和 PNG。
- 保持字体、尺寸、配色、图例和导出参数一致。

## 作图前检查

作图前先确认：

- 数据类型：连续、分类、二值、时间序列、成对数据、重复测量
- 样本分组：分组顺序、参考组、对照组、配对关系
- 缺失值：缺失比例、缺失位置、是否需要剔除或标注
- 异常值：是否需要保留、截断、标记或说明
- 单位和量纲：是否需要标准化、对数转换、中心化
- 统计含义：均值、中位数、比例、相关、回归、显著性

## 图形选择

按数据形态和研究问题选图，而不是先套模板。

- 分组比较：箱线图、小提琴图、柱状图、散点抖动图
- 分布展示：直方图、密度图、箱线图、小提琴图
- 相关关系：散点图、回归图、相关性热图
- 高维降维：PCA、UMAP、聚类图
- 差异结果：火山图、热图、森林图
- 富集结果：条形图、气泡图、富集曲线图
- 生存或模型结果：KM 曲线、ROC 曲线、校准曲线、DCA
- 时间趋势：折线图、分面图、轨迹图

## 风格规则

- 连续变量优先使用单向或发散色阶，并保证方向和数值顺序一致。
- 有明确正负方向的数据优先使用发散色阶。
- 分类变量使用清晰、稳定、可区分的离散配色。
- 同一项目中，相同变量尽量保持相同颜色语义。
- 避免无意义的高饱和大面积填充。
- 使用简洁背景，减少视觉干扰。
- 具体边距、留白、线宽和字号不要在模块正文里写死，保持可配置。

## 用户提供代码时的处理

- 保留统计方法和科学含义。
- 先修复可复现性问题、明显错误和导出问题。
- 把路径、变量名、阈值、配色、图例和尺寸参数化。
- 重复逻辑可以整理为函数，但不要过度抽象。
- 默认保持原项目风格，除非用户要求重新设计。

## 个人模式

个人模式是这个模块的显式覆盖层。

只有用户明确要求个人风格时才启用。

启用后：

- 采用用户指定的配色、字体、字号、间距和导出偏好
- 保持统计内容和科学含义不变
- 保持图形逻辑不变，除非用户要求重新设计

未启用时：

- 使用中性默认值
- 不主动引入个人审美假设

## 导出规则

- 必须同时检查 PDF 和 PNG。
- 画布尺寸、边距、字体和分辨率要一致。
- 标签、单位、坐标轴和图例要适合论文或汇报。
- 多面板图要保持面板编号和排版一致。

## 最小验证

交付前至少检查：

- 连续变量颜色方向是否和数值顺序一致
- 有序变量是否按正确顺序显示
- 分类变量图例顺序是否正确
- 标签、单位、统计量和显著性标注是否正确
- 是否存在裁切、遮挡、过密或过空问题
- PDF 和 PNG 是否都能打开
- 代码是否可从干净的 R 会话复现
