# 组学 QC 分析模块

## 这个 skill 的定位

这个 skill 不再只是“下游前复核一下”。

它的定位是：

1. 承接组学 QC 任务入口。
2. 按 assay 类型进入不同 QC 流程。
3. 检查数据是否适合继续进入差异分析、富集、时序或网络分析。
4. 输出 QC 结论和是否允许进入后续模块的判断。

## 组学分流原则

QC 不能混成一套通用模板，必须按组学分流：

1. `proteomics` 使用蛋白组 QC 逻辑。
2. `bulk-rnaseq` 使用 bulk RNA-seq QC 逻辑。
3. `metabolomics` 使用代谢组 QC 逻辑。

如果 assay 类型不明确，必须停止并追问，不能直接套用蛋白组或转录组习惯。

## 蛋白组 QC 入口规则

当 assay = `proteomics` 时，至少先检查以下内容：

1. 当前输入是原始定量矩阵、log2 后矩阵，还是已经 clean 的蛋白组矩阵。
2. 样本分组和样本名是否能一一对应。
3. 是否已经做过缺失值过滤、样本剔除和缺失值填补。
4. 是否已有 batch 信息，以及 batch 是否可能影响后续差异分析。
5. 是否已有污染、异常样本或低质量样本的判定结论。
6. 当前矩阵是否适合作为 DEP、富集分析、Mfuzz 或 WGCNA 的输入。

如果用户说这是蛋白组数据，QC 流程不能只停留在通用检查层，而要明确进入蛋白组逻辑。

## 必问项

1. 当前数据属于什么组学？
2. 当前输入是 raw、normalized、log2、imputed 还是 clean matrix？
3. 是否已有上游样本剔除结论？
4. metadata 是否完整？
5. 是否存在 batch？
6. 是否已有缺失值处理？

## 目标输出

最少输出：

1. matrix 状态说明
2. metadata 完整性检查
3. batch 是否存在及处理建议
4. 是否允许进入后续分析的结论
5. 若不允许进入后续分析，指出卡住的原因

## 停止项

遇到以下情况必须停止：

1. assay 类型不清楚
2. matrix 状态不清楚
3. metadata 缺失到无法完成样本映射
4. batch、缺失值处理或样本剔除状态不清楚，且它们会影响后续方法选择

## 核心原则

这个 skill 的任务不是替代所有上游原始 QC 脚本，而是：

- 组织和执行组学 QC 流程
- 识别是否还需要回退到更上游的 QC
- 给后续模块一个明确的“可继续 / 不可继续”结论
